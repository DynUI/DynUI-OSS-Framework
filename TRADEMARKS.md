# Trademark Policy

This policy explains how to use the DynUI name, package names, logos, and other
project marks. It is intended to make community use easy while preventing
confusion about who makes, maintains, or endorses a product or service.

This policy is not legal advice. If you have questions about a particular use,
ask first.

## Marks Covered

The marks covered by this policy include:

- `DynUI`;
- `dynui`;
- `@dynui/*` package names;
- DynUI logos, wordmarks, icons, and visual brand assets;
- any confusingly similar names, logos, or visual marks.

Draft project brand assets live in [docs/brand](docs/brand/).

The open source license for the code does not grant permission to use these
marks except as described in this policy.

## Allowed Uses

You may use the DynUI marks without separate permission when your use is
truthful, non-confusing, and limited to referring to the project, including:

- referring to DynUI as the upstream project or framework;
- documenting compatibility with DynUI or `@dynui/*` packages;
- using package names, import paths, CLI names, and repository names as required
  to install, run, test, or integrate the software;
- publishing tutorials, reviews, benchmarks, talks, or examples about DynUI;
- saying that your project is "built with DynUI", "compatible with DynUI", or
  "for DynUI", as long as it is clear that your project is independent.

## Forks And Modified Versions

You may fork, modify, and redistribute the code under its open source license,
but modified versions must avoid implying that they are the official DynUI
project.

Acceptable examples:

- "ExampleUI, based on DynUI";
- "ExampleUI, a fork of DynUI";
- "ExampleUI is compatible with DynUI manifests".

Avoid names such as "DynUI Pro", "DynUI Cloud", "DynUI Enterprise", or
"Official DynUI" unless you have written permission.

## Uses That Need Permission

Please request permission before using the DynUI marks for:

- a company, product, service, hosted platform, consulting business, domain name,
  social account, or event name;
- merchandising, advertising, sponsorships, or paid promotion;
- a logo, app icon, favicon, badge, or visual identity for your own project;
- modified versions of DynUI logos or brand assets;
- any use that could imply endorsement, certification, partnership, or official
  status.

## Prohibited Uses

Do not use the DynUI marks in a way that:

- misleads people about the source, sponsorship, approval, or affiliation of a
  product, service, or project;
- suggests that an unofficial fork, integration, package, or hosted service is
  official;
- registers or attempts to register a confusingly similar trademark, domain,
  package name, repository name, or social handle;
- harms the reputation of the project or uses the marks for malware, phishing,
  spam, or other abusive activity.

## Attribution

When attribution is appropriate, use:

> DynUI and related marks are trademarks of the DynUI project maintainers.

## Permission And Reports

For trademark permission requests or to report confusing use of the marks, open
a GitHub issue or contact the maintainer at **langiakash@gmail.com**.

The maintainers may update this policy as the project, ownership, or branding
changes.
